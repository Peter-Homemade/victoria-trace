# Pre-existing Work and Build Week Attribution

## What predates OpenAI Build Week 2026

The broader Victoria concepts, framing, and prior research predate Build Week.
That pre-existing conceptual work includes interest in long-running agent
continuity, memory provenance, uncertainty, corrections, and the distinction
between historical and currently valid information.

Those concepts provide background and motivation. They are not being represented
as newly invented during Build Week.

## What is being created during Build Week

The Victoria Trace implementation in this repository is being created during
OpenAI Build Week 2026. This includes its repository structure, synthetic demo
history, data model, local Python implementation, command-line workflow, tests,
documentation, and evidence of the correction-to-regression process.

At repository foundation time, no application implementation has been added.
Material work created during Build Week will be recorded in
`BUILD_WEEK_CHANGELOG.md` and in the repository history once commits are reviewed
and authorized.

## Repository boundary

This repository is self-contained. It must not import private material or copy
implementation artifacts from the separate Victoria-Framework repository. Any
future third-party inspiration or reused public code must be documented with its
source and license.
